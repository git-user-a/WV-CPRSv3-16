Marking Vitals as Entered in Error Vitals data values cannot be deleted once they have been saved – if incorrect vitals data have been saved, they must be marked "Entered in Error" and replaced with corrected data. Any data values that have been marked "in error" will continue to be stored in the Vitals database, but will be removed from the patient’s data grid and graph. To mark vitals as "Entered in Error":
  1. In the main Vitals window, select a patient and locate the data value(s) to be corrected. Click the Date/Time column heading –or– select Entered in Error from the File menu. The Entered in Error window opens.
  2. Click the down-arrow button and select the date of the incorrect data from the calendar. A list of vitals data opens, displaying all values for the selected date.
  3. Click an incorrect value to select it, or press CTRL and click to select multiple values. In the Reason section, click the reason for the incorrect entry, then click the Mark as Entered in Error button. A confirmation screen opens.
  4. Click Yes to confirm that the selected value has been entered in error. Click No to select a different value. An information screen appears.
  5. Click OK. All values that have been marked "Entered in Error" are removed from the data grid and graph.
  6. If necessary, enter new vitals data for this patient to replace the incorrect values.
To print a list of all "Entered in Error" vitals data values, see Printing a Report.
